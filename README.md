## HTML - JS - CCS
### Nuestra idea principal fue ver que tan de acuerdo estabamos en cosas generales y cuantas personas pensaban lo mismo, por eso pensamos en el programa de television "100 Argentinos dicen". 
### En nuestro juego queriamos que las preguntas sean de todo tipo pero que tambien sean aleatorias asi no les toca la misma todo el tiempo. Tambien que cada pregunta tenga un limite de 100 segundos asi no se vuelve tan lento, ya que si se termina el tiempo perdes el punto. 

### Para poder hacer esto primero nos dividimos las partes y los tiempos con diagramas de Gantt, asi poder llegar bien a la entrega. Despues hicimos los HTML uno para cada parte del juego y una para el admin. Si ingresas con la cuenta del admin te lleva a un munu donde podes agregar preguntas, borrar respuestas o cambiarlas, tambien se puede borrar el historial de un jugador e incluso asignarle ser administrador a otros jugadores. 

### Despues empesamos a programar el JS, con esto tambien a buscar informacion para poder ponerle un limite de tiempo a el juego. En el JS hicimos los gets generales, los pedidos espesificos (puntajes, partidas, preguntas y sus opciones, usuarios registrados, admins), los posts, puts, deletes.
### Mientras se hacia esto empesamos a hacer el CCS, intentamos que se paresca lo mas parecido posible a la presentacion previa con colores celestes por el color del colegio, un poco de encaje y letra entendible. La intencion con el CCS fue que no sea tan trabajado para que se puedan ver mejor las tablas.
